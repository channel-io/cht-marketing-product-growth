# Conversational Marketing & Sales 시장 가능성

> **목적**: 이 시장이 왜 가능성이 있는지, 채널톡이 진입해야 하는 이유

---

## 1. 시장 시그널: 왜 지금 Conversational인가?

### Drift → Salesloft 인수 (2024.02)

> "시장에 최초이자 유일한 AI-powered Revenue Orchestration Platform을 소개한다" — David Obrand, Salesloft CEO

**인수 배경:**
- Salesloft: 세일즈 engagement 자동화 플랫폼
- Drift: Conversational Marketing/Sales의 창시자

**인수 이유:**
| 이유 | 설명 |
|------|------|
| Sales + Marketing 연결 | 구매자 여정 전체를 하나의 플랫폼에서 |
| 대화 → 파이프라인 | 웹사이트 대화에서 발생한 구매 의도를 즉시 세일즈로 연결 |
| AI Revenue Orchestration | 대화 데이터 + 구매자 행동 → 개인화된 구매 경험 |

**핵심 시사점:**
- Conversational이 GTM(Go-to-Market)의 핵심 레이어로 인정받은 신호
- 단순 리드 수집 도구가 아니라 **Revenue 인프라**로 포지셔닝

### Intercom: Fin as SDR 확장

> "봇이 잘하는 것(리드 정보 수집, 자격검증)은 봇에게, 인간이 잘하는 것(페인포인트 이해, 관계 구축)은 인간에게"

**Fin as SDR 기능:**
- 구매 신호 감지
- 리드 자격검증 (니즈, 목표, 타임라인, 예산, 회사 규모)
- CRM 연동
- 세일즈팀으로 라우팅

**비전:**
- Lead qualification → Onboarding → Support → Success → Upsell
- 전체 고객 여정을 "대화"로 커버

### 글로벌 플레이어 수렴

| 회사 | 움직임 |
|------|--------|
| **Drift** | Salesloft에 인수 (2024) |
| **Intercom** | Fin을 SDR로 확장. CRM 마케팅(이메일/푸시)은 유지만 하고 신규 개발 없음 |
| **Zendesk** | Suite + Sell 통합. CRM 마케팅은 명시적으로 안 함 |
| **Freshworks** | Freshdesk + Freshsales 통합 |

**공통점:** CRM Marketing은 포기/후순위, Conversational Sales로 수렴

## 2. Conversational Marketing/Sales란?

### 개념 정의

```
기존 CRM 마케팅:
고객 데이터 수집 → 세그먼트 → 이메일/푸시 캠페인 → (며칠 후) 반응 확인

Conversational:
방문자 → 실시간 대화 → 의도 파악 → 즉시 전환 (같은 세션)
```

- **Conversational Marketing**: 방문자 → 실시간 대화 → 리드 생성/육성
- **Conversational Sales**: 리드 자격검증 → 세일즈팀 핸드오프 → 계약 전환

### CRM 마케팅과의 핵심 차이

| 구분 | CRM Marketing | Conversational |
|------|---------------|----------------|
| **타이밍** | 비동기 (캠페인 발송 후 대기) | 실시간 (의도 발생 순간 포착) |
| **위치** | 상담 밖 (데이터 추출 → 별도 채널) | 상담 안 (대화 = 전환) |
| **전환 방식** | Nurture → 시간 지연 → 전환 | 의도 순간 포착 → 즉시 전환 |
| **데이터 활용** | 과거 데이터 기반 타겟팅 | 실시간 행동 기반 대응 |

### 패러다임 전환

| 기존 | Conversational |
|------|----------------|
| Form 작성 | 대화로 정보 수집 |
| Nurture 캠페인 | Real-time Qualify |
| Lead 관리 | 의도 순간 포착 + 즉시 전환 |

## 3. CRM 마케팅 서비스들이 후발주자인 이유

### 구조적 한계

| 한계 | 설명 |
|------|------|
| **비동기 설계** | 캠페인 발송 → 반응 대기 구조. 실시간 대응 불가 |
| **과거 데이터 의존** | 과거 행동 기반 타겟팅. "지금 이 순간" 의도 포착 어려움 |
| **채널 분리** | 이메일/푸시는 웹사이트/앱과 별개. 대화 연속성 없음 |
| **전환 지연** | 캠페인 → 클릭 → 랜딩 → 폼 작성 → 영업 연락. 단계마다 이탈 |

### 대화 인프라가 먼저여야 하는 이유

> "고객은 하나의 연속된 대화를 기대한다. 그런데 팀은 도구 사이를 오가며 맥락을 잃는다. 그 단절에서 유망한 거래가 죽는다."

**CRM 마케팅 툴의 딜레마:**
- 실시간 대화 인프라 없이는 Conversational로 전환 불가
- 대화 인프라를 새로 구축해야 함 → 후발주자

**Conversational 플랫폼의 이점:**
- 이미 대화 인프라 보유
- CRM 데이터는 대화의 백엔드로 통합 가능

### CRM의 새로운 역할

> "CRM은 고객을 '아는 방법'이고, Conversational은 고객과 '돈을 버는 방법'이다"

| CRM 역할 | 설명 |
|----------|------|
| ❌ 전면 제품 | 더 이상 고객 접점 아님 |
| ⭕ 인프라/백엔드 | 고객 상태, 히스토리, 규칙, 성과 집계 |

## 4. 채널톡이 진입 시 얻는 이점

### 이미 보유한 해자

| 자산 | 설명 |
|------|------|
| **실시간 대화 인프라** | 웹챗, 인앱 메시지 이미 구축 |
| **상담 자동화 경험** | ALF(AI 상담원) 기술 보유 |
| **"지금 의도" 순간 점유** | 고객이 문제/관심이 생긴 순간 이미 채널톡에 있음 |
| **고객 데이터** | 상담 히스토리, 행동 데이터 축적 |

### 경쟁 우위

| vs | 채널톡 이점 |
|----|------------|
| **CRM 마케팅 툴** | 대화 인프라 없음. 새로 구축해야 함 |
| **순수 Chatbot 툴** | 상담 깊이 없음. CS 경험 부족 |
| **글로벌 CS 툴** | 한국/일본 시장 이해, 로컬 최적화 |

### 자연스러운 확장 경로

```
현재: CS 상담 (문제 해결)
  ↓
확장: Pre-sales 상담 (구매 전환)
  ↓
확장: Post-sales (Upsell, Expansion)
```

상담 해자 위에서 자연스럽게 확장 가능

**지역별:**
- North America: 39.5% ($5.87B)

| 지표 | 수치 |
|------|------|

**97%의 리테일러가 내년 AI 투자 증가 계획** (NVIDIA)

### 확장 방향

| 단계 | 영역 | 설명 |

## 6. 결론: 왜 이 시장인가?

### 시장 검증

1. **글로벌 플레이어 수렴**: Drift 인수, Intercom 확장, Zendesk/Freshworks 통합
2. **시장 규모**: $14.9B → $63.4B (CAGR 15.6%)
3. **ROI 검증**: 370% (3년), 전환율 23% ↑, 비용 30% ↓

### 채널톡 적합성

1. **해자 활용**: 대화 인프라, 상담 자동화, "지금 의도" 순간 점유
2. **자연스러운 확장**: CS → Pre-sales → Post-sales
3. **경쟁 우위**: CRM 툴 대비 대화 인프라, 순수 챗봇 대비 상담 깊이

### 전략적 포지션

> **CRM은 확장하지 않는다. Conversational을 주력 축으로 삼는다.**
> 리소스가 부족할수록 이 선택은 더 합리적이다.
