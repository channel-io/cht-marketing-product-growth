# 개선 방안 제안

## 문제 요약

마케팅 역량이 없는 카페24/아임웹 기반 소규모 이커머스 사업자가, "마케팅을 뭘 해야 할지 몰라서" 신규 전환·상품 추천 마케팅을 시작하지 못하고 있다.

### 원인
- "뭘 해야 할지 모름" (기획 역량 부재)
- 언제, 누구에게, 무슨 내용을, 어떤 채널로 보내야 하는지 전부 모름

### UT를 통해 검증된 사항
- 레시피가 있으면 완료 가능, 없으면 포기
- 캠페인 편집 화면에서 설정 항목 조합 방법을 모름
- 용어 장벽 (세그먼트, 트리거, 퍼블리시 등 11개 용어 이해 못함)
- 30분 이상 걸리면 포기

## 경쟁사 벤치마킹

### 사용성 수준 비교

| 서비스 | 접근 방식 | 사용자 액션 |
|--------|----------|-------------|
| Omnisend | 27개 템플릿에서 선택 → 커스텀 → 활성화 | 템플릿 선택 + 약간의 설정 |
| **데이터라이즈** | **연동하면 AI가 자동화** | **연동만 하면 끝** |
| 채널톡 (AS-IS) | 레시피 찾아서 설정 | 탐색 → 선택 → 설정 → 퍼블리시 |

### 시사점
- 고객 입장에서 가장 편한 건 "연동만 하면 끝"
- 데이터라이즈가 이미 이 방향으로 가고 있음
- UX만으로 차별화하기 어려움 → 사용성은 따라가고, 기능으로 차별화 필요

## 개선 방향: 자동 설정

### 핵심 아이디어
- 빌더사 연동 완료 시 → 핵심 프리셋이 **설정된 상태로 준비**
- 사용자는 "켜기"만 하면 바로 시작
- 메시지 내용, 타이밍 등 이미 채워져 있음

### UX 사용성 수준

| 수준 | 설명 | 사용자 액션 |
|------|------|-------------|
| Lv1 (AS-IS) | 레시피 찾아서 설정 | 탐색 → 선택 → 설정 → 퍼블리시 |
| Lv2 | 퍼널별 프리셋 켜기/끄기 | 퍼널 이해 → 선택 → 켜기 |
| **Lv3 (목표)** | **연동하면 자동 설정, 켜기만** | **켜기 버튼 하나** |

### 자동 활성화 vs 자동 설정

| 방식 | 설명 | 장단점 |
|------|------|--------|
| 자동 활성화 | 연동하면 바로 작동 | 과금 우려, 원치 않는 발송 불안 |
| **자동 설정** | 준비만 되어 있고, 사용자가 켬 | 과금 통제권 보장, 안심 |

### AS-IS → TO-BE 플로우

**AS-IS (현재)**
```
마케팅 메뉴 → 레시피 목록 → 레시피 선택 → 내용 보충 → 퍼블리시
         ↑         ↑              ↑           ↑
      "뭐가 있지?"  "이게 뭐지?"   "뭘 써야 하지?" "이거 눌러도 되나?"
```

**TO-BE (목표)**
```
빌더사 연동 완료 → 마케팅 홈 진입 → "준비된 마케팅" 확인 → 켜기
                                          ↑
                              "오, 이미 다 세팅되어 있네?"
                              "켜기만 하면 되는 거야?"
```

### 핵심 가치
1. 설정 부담 제거: 메시지, 타이밍 등 이미 채워져 있음
2. 의사결정 최소화: "뭘 써야 하지?" 고민 불필요
3. 과금 통제권 보장: 사용자가 켜기 전까지 비용 발생 안 함

## 화면 설계 의도

> 시각적 시안은 프로토타입에서 확인
> - 마케팅 홈 (연동 후): `/prototype/marketing-home.html`
> - 마케팅 홈 (연동 전): `/prototype/marketing-home-no-integration.html`
> - 캠페인 리스트: `/prototype/campaign-list.html`

### 설득 구조

1. "고객 여정은 이렇게 생겼어요" (맥락)
2. "당신 쇼핑몰은 여기서 이탈이 많아요" (문제 인식)
3. "이 마케팅을 켜면 이 단계를 개선할 수 있어요" (해결책)

### 화면 구성 의도

| 요소 | 의도 |
|------|------|
| 고객 여정 시각화 | 마케팅 용어 없이 퍼널 개념 전달 |
| 실제 내 쇼핑몰 숫자 | "내 얘기구나" 몰입 |
| 업계 평균 비교 | "나 낮구나" 문제 인식 |
| 취약 단계 하이라이트 | 우선순위 명확, "여기부터 해야겠다" |
| AI 추천 뱃지 | "내가 고민 안 해도 되겠다" 신뢰 |
| 소셜 프루프 (사용 채널 수, 평균 전환율) | "다른 사람도 쓰고 있구나" 안심 |
| 단계별 1개 메인 추천 | 선택 부담 제거 |
| [+ 더 보기] → 캠페인 리스트 (퍼널 필터 적용) | 필요시 추가 옵션 확인 가능 |
| 예상 효과 | "켜면 좋아지겠다" 동기 부여 |
| 메시지 미리보기 | 빈칸 채울 필요 없음 확인 |
| [켜기] 버튼 하나 | 복잡한 설정 과정 없음 |
| 쉬운 말 사용 | "담아두고 안 산 고객" (세그먼트 X) |

### 기대하는 사용자 반응 (민지 페르소나)

> "오, 내 쇼핑몰 데이터로 보여주네"
> "방문은 많은데 가입이 적구나... 15%밖에 안 되네"
> "업계 평균이 25%라고? 그럼 나 낮은 거네"
> "이거 켜면 +5~10% 오를 수 있다고? 켜볼까"

### 네비게이션 구조 (탭 정리)

**AS-IS (5개)**
- 홈, 일회성, 캠페인, 마케팅 레시피, 사용 가이드

**TO-BE (3개)**
- 홈: 추천 마케팅 켜기 (시작점)
- 캠페인: 켜놓은 마케팅 관리 (일회성 통합)
- 사용 가이드

**제거 사유:**
- 마케팅 레시피: 홈에서 프리셋 제공하므로 불필요
- 일회성: 캠페인에 통합

## 자동 설정 대상 프리셋

### 이커머스 CRM 퍼널 (일반적)

```
방문 → 회원가입 → 첫 구매 → 재구매 → 충성고객
        ↓           ↓         ↓         ↓
      [이탈]      [이탈]     [이탈]    [이탈]
```

### 현재 범위 (신규 전환, 상품 추천)

| 퍼널 단계 | 자동 설정 프리셋 | 기대 효과 |
|-----------|------------------|-----------|
| 방문 → 회원가입 | 가입 유도 팝업 + 쿠폰 | 리드 확보 |
| 회원 → 첫 구매 | 장바구니 이탈 리마인드 | 신규 전환 |
| 상품 추천 | 최근 본 상품 추천 | 구매 유도 |

## 전제 조건

- 카페24/아임웹 빌더사 연동이 되어 있는 고객사 대상
- 연동이 안 된 경우 이커머스 이벤트(장바구니 담기 등) 사용 불가
- 연동 안 된 상태에서도 일부 기능(회원가입 유도)은 사용 가능

---

## 기능적 차별화 (별도 논의 필요)

UX 사용성은 경쟁사 수준으로 따라가되, 채널톡만의 차별점은 기능으로 확보 필요.

### 채널톡 고유 자산
- 실시간 채팅 채널
- 상담 대화 데이터

### 차별화 후보 (추후 논의)
1. 상담 기반 마케팅 트리거 (예: "이 상품 문의한 고객"에게 자동 팔로업)
2. 채팅 기반 마케팅 메시지 (팝업이 아닌 채팅창으로 전달, 양방향 가능)
3. 상담 중 실시간 추천 (상담원에게 "이 고객에게 이 상품 추천하세요" 제안)

## 다음 단계

### 팀 내 논의 필요
1. 자동 설정할 프리셋 목록 확정
2. 프리셋별 기본값 설정 (메시지 내용, 타이밍, 채널 등)
3. 켜기 후 성과 확인 화면 설계

### 검증 필요
- "준비된 마케팅" 문구가 와닿는지 사용자 반응 확인
- 미리보기 → 켜기 플로우가 충분히 쉬운지 검증
- 연동 안 된 사용자의 연동 전환율
