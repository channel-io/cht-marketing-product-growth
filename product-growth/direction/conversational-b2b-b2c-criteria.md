# Conversational B2B vs B2C 선택 기준

> **목적**: Conversational Marketing/Sales 방향에서 B2B vs B2C 중 우선 타겟 판단

---

## 현재 확인된 사항

| 항목 | 상태 | 내용 |
|------|------|------|
| Sales ALF 방향 | ✅ 확인됨 | **B2B IS (Inside Sales)** 타겟, 데모 개발 중 |
| 고객 구성 데이터 | 🔲 미확인 | 데이터 요청 필요 |

### 검증 필요 사항

Sales ALF의 B2B IS 방향이 **고객 데이터 기반 선택이 아님** → 실제 고객 구성과 맞는지 검증 필요

```
Sales ALF 방향 (B2B IS)
        ↓
실제 고객이 B2C 다수라면?
        ↓
방향 재검토 필요할 수 있음
```

---

## 판단 기준 (우선순위)

### 1순위: Sales ALF 방향

| 이유 | 설명 |
|------|------|
| 전사 전략 정합 | 회사가 베팅하는 방향과 일치해야 시너지 |
| 리소스 충돌 방지 | 다른 방향 가면 제품팀과 마찰 |
| 빠른 실행 | 이미 만들고 있는 것 위에 올라타면 속도 ↑ |

**현재 상태**: B2B IS → B2B 방향 시사

### 2순위: 현재 고객 구성

| 이유 | 설명 |
|------|------|
| 검증된 PMF | 고객이 많은 쪽 = 이미 잘하는 영역 |
| 확장 용이 | 기존 고객에게 추가 기능 판매가 쉬움 |
| 데이터 기반 | 감이 아닌 숫자로 판단 |

**현재 상태**: 🔲 데이터 확인 필요

### 판단 로직

| 시나리오 | 결론 |
|----------|------|
| Sales ALF (B2B) = 고객 다수 (B2B) | ✅ B2B로 확정 |
| Sales ALF (B2B) ≠ 고객 다수 (B2C) | ⚠️ 추가 논의 필요 |

## 필요 데이터

### 분류 기준

| 분류 | 정의 |
|------|------|
| **B2B vs B2C** | 고객사의 판매 대상 (기업 vs 소비자) |
| **SMB / Mid / Ent** | 고객사 규모 (연 매출 기준: ~100억 / 100억~2,500억 / 2,500억~) |
| **버티컬** | 이커머스, SaaS, 금융, 교육 등 |

### 필요 지표 (세그먼트별)

- 고객 수
- MRR

> 범위: 마케팅 기능 사용 고객만이 아닌 **전체 채널톡 고객**

## 버티컬별 Conversational Sales 기회 추론

### 기회 유형 구분

| 유형 | 설명 | 예시 |
|------|------|------|
| **Pre-sales 대응** | 고객이 먼저 문의 → 응대하며 전환 | "이 제품 사이즈 어떻게 되나요?" |
| **Proactive 세일즈** | 우리가 먼저 접근 → 전환 유도 | 이탈 감지 → 상품 추천 → 구매 유도 |
| **Post-sales (CS)** | 구매 후 문의 응대 | 배송, 반품, 결제 문의 |

### 버티컬별 기회

| 버티컬 | Pre-sales 대응 | Proactive 세일즈 | Conversational Sales 기회 |
|--------|---------------|-----------------|-------------------------|
| B2B SaaS | ⭕ 높음 | △ | ⭕ 높음 |
| 고가 이커머스 (가구, 가전) | ⭕ 높음 | ⭕ | ⭕ 높음 |
| 금융/보험 | ⭕ 높음 | △ | ⭕ 높음 |
| 교육 | ⭕ 높음 | △ | ⭕ 높음 |
| 일반 이커머스 (패션, 잡화) | △ 낮음 | ⭕ 높음 (Rep AI 패턴) | ⭕ 있음 |
| 구독 서비스 | △ 낮음 | ⭕ (업그레이드 유도) | ⭕ 있음 |

> **Proactive 세일즈**: 이탈 방지, 상품 추천, 장바구니 복구 등 → 일반 이커머스에서도 기회 있음

→ 버티컬 분포 데이터로 Conversational Sales 기회 크기 추론 가능

## 마켓 사이즈별 기회 매트릭스

B2B/B2C와 마켓 사이즈(SMB/Mid/Ent)를 조합한 전략 매트릭스:

### 마켓 사이즈 정의

| 세그먼트 | 연 매출 | 특징 |
|----------|---------|------|
| SMB | ~100억 | 세일즈팀 없거나 소규모 |
| Mid-Market | 100억~2,500억 | 세일즈팀 있음, 적당한 복잡도 |
| Enterprise | 2,500억~ | 전담 SDR/세일즈팀, 복잡한 구매 프로세스 |

### 조합별 Conversational Sales 기회

| | SMB | Mid-Market | Enterprise |
|------|-----|------------|------------|
| **B2B** | △ 세일즈팀 없음, 단순 구매 | ⭕ **Sweet spot** | △ 이미 전담 SDR 있음 |
| **B2C** | △ 트래픽 적음, 비용 정당화 어려움 | ⭕ Rep AI 패턴 적용 가능 | △ 자체 구현 선호 |

### 시사점

- **B2B + Mid-Market**: Conversational Sales (B2B IS) 최적 타겟
- **B2C + Mid-Market**: Proactive 세일즈 (Rep AI 패턴) 가능
- 회사의 Mid-Market 집중 방향과 일치하면 시너지

## 시나리오별 전략

### 배경: 크로스셀 전략

> 참고: [부가 기능 성장 패턴](../references/addon-growth-patterns.md)

성공 사례(HubSpot, Intercom, Slack, Notion)의 공통점:
- 핵심 제품 고객에게 부가 기능을 **크로스셀**
- 업셀(같은 제품 더 많이)이 아닌 크로스셀(다른 기능 추가)

**채널톡 적용**: 기존 상담 고객에게 → Conversational Sales 기능 크로스셀

### 시나리오별 액션

| 고객 구성 | 결론 | 액션 |
|----------|------|------|
| **B2B가 많거나 비슷** | ✅ 현재 방향과 일치, TAM 큼 | B2B 고객 전체 대상 크로스셀 캠페인 |
| **B2B가 적음** | ⚠️ 방향은 B2B 유지, TAM 작음 | B2B 고객 집중 타겟 크로스셀 (소수 정예) |

### 시나리오 1: B2B가 많거나 비슷

```
전체 고객 중 B2B 비중 높음
        ↓
Conversational Sales 기회 큼
        ↓
B2B 고객 전체 대상 크로스셀
```

**액션:**
- B2B 고객 세그먼트 전체에 Conversational Sales 소개
- 버티컬별 맞춤 메시지 (SaaS, 금융, 교육 등)
- Sales ALF 데모/베타 참여 유도

### 시나리오 2: B2B가 적음

```
전체 고객 중 B2C 비중 높음
        ↓
B2B 고객은 소수지만 기회는 높음
        ↓
소수 B2B 고객에 집중 크로스셀
```

**액션:**
- B2B 고객 리스트업 (소수 정예)
- 1:1 맞춤 접근 (규모가 작으니 가능)
- 성공 사례 만들기 → 이후 B2B 고객 확대 전략
