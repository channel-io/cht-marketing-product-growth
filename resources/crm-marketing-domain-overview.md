# CRM 마케팅 도메인 오버뷰

## 1. 정의

**CRM 마케팅**은 고객 데이터를 기반으로 개인화된 메시지를 적시에 전달하여 고객 관계를 구축하고 비즈니스 성과를 만드는 마케팅 방식이다.

| 일반 마케팅 | CRM 마케팅 |
|-------------|------------|
| 불특정 다수 대상 | 식별된 고객 대상 |
| 채널/매체 중심 | 고객 중심 |
| 노출/도달 지표 | 전환/LTV 지표 |
| 광고비 기반 | 데이터 기반 |

---

## 2. 역사와 진화

```
1990s          2000s          2010s          2020s
  │              │              │              │
  ▼              ▼              ▼              ▼
Database     Email          Marketing      Real-time
Marketing    Marketing      Automation     Personalization
  │              │              │              │
고객 DB 구축   이메일 대량발송   자동화 시나리오   AI/실시간 개인화
                              리드 스코어링    옴니채널 통합
                                             CDP 등장
```

**진화의 핵심 흐름:**
- 배치(Batch) → 실시간(Real-time)
- 채널 분리 → 옴니채널 통합
- 규칙 기반 → AI/ML 기반
- 캠페인 중심 → 고객 여정 중심

---

## 3. 핵심 구성요소

### 3.1 세그먼테이션 (Segmentation)
고객을 의미 있는 그룹으로 나누는 것

- **속성 기반**: 나이, 지역, 가입일
- **행동 기반**: 구매 이력, 앱 사용 패턴
- **RFM**: Recency(최근성), Frequency(빈도), Monetary(금액)
- **예측 기반**: 이탈 확률, 구매 확률

### 3.2 개인화 (Personalization)
개별 고객에게 맞춤 경험 제공

| 수준 | 예시 |
|------|------|
| 이름 삽입 | "홍길동님, 안녕하세요" |
| 세그먼트별 | VIP 고객에게 다른 혜택 |
| 1:1 추천 | 개인별 상품 추천 |
| 실시간 | 행동 즉시 반응 |

### 3.3 자동화 (Automation)
트리거 기반으로 메시지/액션을 자동 실행

```
[이벤트 발생] → [조건 확인] → [액션 실행] → [결과 측정]
    │               │              │
 장바구니 이탈    24시간 경과?    리마인드 발송
 회원가입        첫 구매 완료?    온보딩 시작
 비활성 30일     VIP 여부?       이탈방지 쿠폰
```

### 3.4 채널 (Channels)
고객에게 도달하는 접점

| 채널 | 특성 | 적합 용도 |
|------|------|-----------|
| 이메일 | 긴 콘텐츠, 저비용 | 뉴스레터, 프로모션 |
| 푸시 | 즉시성, 짧은 메시지 | 긴급 알림, 리마인드 |
| 인앱 | 앱 사용 중 노출 | 온보딩, 기능 안내 |
| SMS | 높은 도달률 | 인증, 긴급 공지 |
| 카카오톡 | 한국 필수 채널 | 주문/배송, CS |
| 웹 푸시 | 앱 없이 도달 | 재방문 유도 |

## 4. 데이터

### 4.1 수집하는 데이터

| 유형 | 예시 |
|------|------|
| **프로필** | 이름, 이메일, 연락처, 가입일 |
| **속성** | 나이, 성별, 지역, 멤버십 등급 |
| **행동 이벤트** | 페이지뷰, 클릭, 구매, 검색 |
| **거래** | 주문, 결제, 환불 |
| **캠페인 반응** | 오픈, 클릭, 전환 |
| **기기/환경** | 디바이스, OS, 브라우저 |

### 4.2 데이터 활용 흐름

```
[수집] → [통합] → [분석] → [세그먼트] → [액션] → [측정]
  │        │        │          │          │        │
SDK      CDP     고객 360    타겟 정의   캠페인    성과 분석
API      통합ID   행동 분석   조건 설정   발송      A/B 테스트
```

## 5. 기술 아키텍처

### 5.1 주요 시스템 구성

```
┌─────────────────────────────────────────────────────────┐
│                      데이터 소스                          │
│  (웹/앱 SDK, 서버 API, 3rd Party 연동)                    │
└─────────────────────────┬───────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────┐
│                 CDP (Customer Data Platform)             │
│  - 데이터 수집/통합                                        │
│  - 고객 프로필 통합 (Identity Resolution)                  │
│  - 세그먼트 생성                                          │
└─────────────────────────┬───────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────┐
│              CEP (Customer Engagement Platform)          │
│  - 캠페인 오케스트레이션                                   │
│  - 자동화 워크플로우                                       │
│  - 개인화 엔진                                            │
└─────────────────────────┬───────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────┐
│                     채널 (Channels)                      │
│  이메일 | 푸시 | SMS | 인앱 | 카카오 | 웹푸시              │
└─────────────────────────────────────────────────────────┘
```

### 5.2 핵심 기술 개념

| 개념 | 설명 |
|------|------|
| **Identity Resolution** | 여러 기기/채널의 고객을 하나로 통합 |
| **Event Streaming** | 실시간 이벤트 수집/처리 (Kafka 등) |
| **Real-time Trigger** | 이벤트 발생 즉시 반응 |
| **Journey Orchestration** | 복잡한 고객 여정 설계/실행 |
| **A/B Testing** | 메시지/타이밍 등 실험 |

## 6. 성과 측정

### 6.1 캠페인 레벨 지표

| 지표 | 설명 |
|------|------|
| **발송률** | 발송 성공 / 전체 타겟 |
| **오픈률** | 오픈 / 발송 성공 |
| **클릭률 (CTR)** | 클릭 / 오픈 |
| **전환율** | 전환 / 클릭 |
| **수신거부율** | Opt-out / 발송 |

### 6.2 비즈니스 레벨 지표

| 지표 | 설명 |
|------|------|
| **LTV** | 고객 생애 가치 |
| **Retention Rate** | 재방문/재구매율 |
| **Churn Rate** | 이탈률 |
| **ARPU** | 유저당 평균 매출 |
| **CAC:LTV Ratio** | 획득비용 대비 생애가치 |

### 6.3 측정의 어려움

- **Attribution**: 어떤 캠페인이 전환에 기여했는지
- **Incrementality**: 캠페인이 없었어도 전환했을 고객 구분
- **Cross-device**: 여러 기기에서의 행동 연결

## 7. 시장과 경쟁

### 7.1 시장 규모

- 글로벌 마케팅 자동화 시장: ~$7B (2025) → ~$18B (2030)
- 연평균 성장률: 12-15%

### 7.2 주요 플레이어 포지셔닝

```
                    Enterprise
                        │
         Oracle ────────┼──────── Salesforce
         SAP            │         Adobe
                        │
   Complexity ──────────┼────────────────── Ease of Use
                        │
         Braze ─────────┼──────── HubSpot
         Iterable       │         ActiveCampaign
                        │         Klaviyo
                       SMB
```

### 7.3 경쟁 구도 유형

| 포지션 | 전략 | 예시 |
|--------|------|------|
| **All-in-one** | CRM+마케팅+세일즈 통합 | Salesforce, HubSpot |
| **Best-of-breed** | 특정 영역 최고 | Braze(모바일), Klaviyo(이커머스) |
| **버티컬 특화** | 산업별 맞춤 | Yotpo(이커머스 리뷰) |
| **가격 경쟁** | SMB 저가 공략 | Mailchimp, Sendinblue |

## 핵심 요약

> **CRM 마케팅 = 데이터 × 세그먼트 × 개인화 × 자동화 × 채널**

1. **고객 데이터**를 모아서
2. 의미 있는 **세그먼트**로 나누고
3. **개인화**된 메시지를
4. 적절한 **타이밍**에 (자동화)
5. 적합한 **채널**로 전달하여
6. **성과를 측정**하고 개선한다
