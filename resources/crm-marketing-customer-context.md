# CRM 마케팅 툴 고객사의 컨텍스트와 니즈

> **핵심 질문**: 왜 CRM 마케팅이라는 하나의 범주 안에서도 툴의 방향성이 여러 갈래로 나뉘는가?
>
> **답**: 고객사의 컨텍스트가 다르기 때문에 니즈가 다르고, 니즈가 다르기 때문에 툴이 분화된다.

---

## 고객사 컨텍스트의 두 가지 축

| 구분 | 설명 | 예시 |
|------|------|------|
| **영속적 속성** | 쉽게 바뀌지 않는 본질적 특성 | 비즈니스 모델, 산업, 타겟 고객 유형 |
| **변화하는 속성** | 시간에 따라 진화하는 특성 | 마케팅 성숙도, 팀 규모, 기술 역량, 성장 단계 |

# Part 1. 영속적 속성

## 1.1 비즈니스 모델: B2B vs B2C

가장 근본적인 분기점. 세일즈 사이클, 의사결정 구조, 핵심 지표가 완전히 다르다. [^2] [^3]

### 비교

| 차원 | B2B | B2C |
|------|-----|-----|
| **구매 의사결정** | 복수 이해관계자, 긴 검토 과정 | 개인, 빠른 결정 (충동구매 가능) |
| **세일즈 사이클** | 수 주 ~ 수 개월 | 즉시 ~ 수 일 |
| **거래 단가** | 높음 (고가치) | 낮음 (저가치, 다빈도) |
| **핵심 페인포인트** | 리드 생성 & 전환 (Lifecycle 관리) | 고객 유지 & 이탈 방지 (Retention) |
| **핵심 지표** | MQL, SQL, 파이프라인, Win Rate | LTV, Retention Rate, 재구매율 |
| **데이터 유형** | 회사 규모, 산업, 담당자 직책, 니즈 | 나이, 성별, 구매 이력, 행동 패턴 |
| **주요 채널** | 이메일, 콘텐츠, 세일즈 연계 | 이메일, 푸시, SMS, 인앱, 리타게팅 |

### 니즈 차이

**B2B 고객사의 니즈:**
- 리드 스코어링: 어떤 리드가 구매 가능성이 높은가?
- 리드 너처링: 긴 세일즈 사이클 동안 관계 유지
- CRM 연동: 세일즈팀과 데이터 공유
- 계정 기반 마케팅 (ABM): 개별 기업 단위 타겟팅
- 콘텐츠 자동화: 화이트페이퍼, 웨비나, 케이스 스터디 배포

**B2C 고객사의 니즈:**
- 대량 개인화: 수십만~수백만 고객에게 맞춤 메시지
- 멀티채널 자동화: 이메일 + 푸시 + SMS + 인앱 통합
- 리텐션 마케팅: 재구매 유도, 이탈 방지
- 리타게팅: 장바구니 이탈, 브라우징 이탈 대응
- 실시간 반응: 행동 즉시 트리거

## 1.2 산업/버티컬: 이커머스 vs SaaS vs 모바일 앱

같은 B2C라도 비즈니스 유형에 따라 고객 여정과 핵심 지표가 다르다.

### 이커머스

**고객 여정의 특징:**
- 상품 탐색 → 장바구니 → 결제 → 배송 → 재구매
- 구매 빈도가 핵심 (반복 구매 유도)
- 상품 카탈로그와 재고 연동 필요

**핵심 니즈:**
- 장바구니 이탈 리마인드
- 상품 추천 (협업 필터링, 콘텐츠 기반)
- 가격 할인/쿠폰 자동화
- 재고 기반 캠페인 (품절 임박, 재입고 알림)
- 로열티/멤버십 프로그램 연동
- 리뷰 요청 자동화

**핵심 지표:**
- 전환율, AOV (평균 주문 금액), 재구매율, LTV

---

### SaaS (구독 서비스) [^7]

**고객 여정의 특징:**
- 가입 → 온보딩 → 활성화 → 유료 전환 → 확장 → 갱신/이탈
- 제품 사용 행동이 핵심 데이터
- Trial → Paid 전환이 중요

**핵심 니즈:**
- 행동 기반 메시징: 특정 기능 사용/미사용 시 안내
- 온보딩 자동화: 핵심 기능까지 도달하도록 유도
- 인앱 가이드: 툴팁, 모달, 체크리스트
- 이탈 예측: 사용량 감소 시 선제 대응
- 업그레이드/업셀: 상위 플랜 제안
- NPS/피드백 수집 자동화

**핵심 지표:**
- Activation Rate, Trial-to-Paid, MRR, Churn Rate, NRR

---

### 모바일 앱

**고객 여정의 특징:**
- 설치 → 첫 실행 → 핵심 액션 → 습관화 → 유료화/광고 노출
- 앱 삭제/비활성이 쉬움 → 리텐션 극히 중요
- 푸시 알림이 핵심 채널

**핵심 니즈:**
- 푸시 알림 최적화: 발송 시점, 빈도, 개인화
- 인앱 메시지: 앱 사용 중 컨텍스트에 맞는 안내
- 딥링크: 특정 화면으로 직접 유도
- 리인게이지먼트: 비활성 유저 복귀 유도
- A/B 테스트: 메시지, 타이밍, 크리에이티브
- 앱 이벤트 분석: 퍼널, 코호트, 리텐션 커브

**핵심 지표:**
- DAU/MAU, D1/D7/D30 Retention, ARPU, LTV

---

### 산업별 니즈 비교 요약

| 차원 | 이커머스 | SaaS | 모바일 앱 |
|------|----------|------|-----------|
| **핵심 목표** | 구매 전환, 재구매 | 활성화, 이탈 방지 | 리텐션, 습관화 |
| **주요 채널** | 이메일, SMS, 웹 | 이메일, 인앱 | 푸시, 인앱 |
| **데이터 초점** | 구매/탐색 행동, 상품 선호 | 제품 사용 행동, 기능 채택 | 앱 이벤트, 세션 |
| **자동화 초점** | 장바구니, 추천, 프로모션 | 온보딩, 행동 트리거, 이탈 예측 | 푸시 최적화, 리인게이지먼트 |
| **연동 필요** | 상품 카탈로그, 재고, 주문 | 제품 이벤트, 플랜/빌링 | 앱 SDK, 딥링크 |

## 1.3 타겟 고객 유형

마케팅 대상이 누구인지에 따라 채널과 메시지 전략이 달라진다.

| 타겟 | 특성 | 채널 선호 | 메시지 톤 |
|------|------|-----------|-----------|
| **Z세대** | 모바일 네이티브, 짧은 집중력 | 푸시, 인앱, 소셜 | 캐주얼, 비주얼 중심 |
| **밀레니얼** | 디지털 친숙, 가치 중시 | 이메일, 푸시, SMS | 개인화, 투명성 |
| **X세대/베이비붐** | 이메일 선호, 신뢰 중시 | 이메일, SMS | 정보 중심, 신뢰 구축 |
| **비즈니스 의사결정자** | 시간 부족, ROI 중시 | 이메일, 링크드인 | 전문적, 가치 제안 명확 |

# Part 2. 변화하는 속성

하나의 기업도 시간이 지나면서 성장하고, 성숙해지고, 니즈가 진화한다.

## 2.1 마케팅 성숙도 (Marketing Maturity) [^4]

### 성숙도 5단계 모델

```
[Ad Hoc] → [Defined] → [Integrated] → [Optimized] → [Transformational]
   1           2            3             4               5
```

| 단계 | 특징 | 마케팅 활동 | 툴 니즈 |
|------|------|-------------|---------|
| **1. Ad Hoc** | 즉흥적, 비체계적 | 필요할 때만 발송, 일관성 없음 | 단순 이메일 발송 |
| **2. Defined** | 프로세스 수립 | 정기 캠페인, 기본 세그먼트 | 이메일 마케팅 + 기본 세그먼트 |
| **3. Integrated** | 시스템 연결 | 데이터 통합, 자동화 시작 | 마케팅 자동화 + CRM 연동 |
| **4. Optimized** | 데이터 기반 의사결정 | A/B 테스트, 예측 모델 | 고급 자동화 + 분석 |
| **5. Transformational** | 마케팅이 성장 엔진 | AI 기반 개인화, 실시간 최적화 | CDP + AI + 옴니채널 |

### 단계별 전환의 가치

| 전환 | 얻는 가치 | 소요 기간 (일반적) |
|------|-----------|-------------------|
| 1→2 | **일관성**: 낭비 감소, 패턴 발견 | 1~3개월 |
| 2→3 | **효율성**: 수작업 제거, 데이터 연결 | 6~12개월 |
| 3→4 | **정밀성**: 예측 가능성, 최적화 | 1~2년 |
| 4→5 | **경쟁 우위**: 마케팅이 차별화 요소 | 지속적 투자 |

### 성숙도별 핵심 니즈

**초기 (1-2단계):**
- 쉬운 설정, 빠른 시작
- 템플릿 제공
- 기본적인 성과 확인
- 저렴한 가격

**중기 (3단계):**
- 데이터 연동 (CRM, 제품, 주문 등)
- 자동화 워크플로우
- 세그먼트 고도화
- 멀티채널 지원

**후기 (4-5단계):**
- 고급 분석, 어트리뷰션
- AI/ML 기반 예측
- 실시간 개인화
- 커스터마이징, API

## 2.2 기업 성장 단계 [^5] [^6]

### 스타트업 → 스케일업 → 엔터프라이즈

```
[Startup] ────────→ [Scaleup] ────────→ [Enterprise]
 ~40명                50~1000명            1000명+
 PMF 탐색             성장 가속            효율화/최적화
```

| 단계 | 조직 특성 | 마케팅 팀 | 툴 니즈 |
|------|-----------|-----------|---------|
| **Startup** | 작은 팀, 자원 제한, 실험 중심 | 1~2명 제너럴리스트, 외부 의존 | 올인원, 저비용, 쉬운 사용 |
| **Scaleup** | 급성장, 프로세스 구축 | 5~15명, 전문화 시작 | 확장성, 자동화, 채널 확대 |
| **Enterprise** | 대규모, 복잡한 조직 | 부서별 분화 (브랜드, 그로스, 프로덕트 마케팅) | 엔터프라이즈급 보안, 커스텀, 통합 |

### 성장 단계별 핵심 니즈

**Startup:**
- PMF 검증 단계 → 빠른 실험 필요
- 예산 제한 → 무료/저가 티어 중요
- 인력 부족 → 쉬운 사용성, 올인원
- "일단 돌아가게" 하는 것이 우선

**Scaleup:**
- 검증된 모델 확장 → 자동화/효율화 필요
- 팀 성장 → 협업 기능, 권한 관리
- 채널 다양화 → 멀티채널 지원
- 데이터 증가 → 분석/세그먼트 고도화

**Enterprise:**
- 복잡한 조직 → 워크플로우, 승인 프로세스
- 보안/컴플라이언스 → SOC2, GDPR, 감사 로그
- 레거시 연동 → 다양한 시스템 통합
- 글로벌 → 다국어, 타임존, 지역별 규정

## 2.3 마케팅 팀 규모와 구성

### 팀 규모에 따른 니즈 변화

| 팀 규모 | 구성 | 니즈 |
|---------|------|------|
| **1~2명** | 제너럴리스트 | 올인원, 쉬운 UI, 템플릿, 시간 절약 |
| **3~5명** | 역할 분화 시작 | 협업 기능, 기본 권한 관리 |
| **6~15명** | 채널별/기능별 전문화 | 고급 권한, 워크플로우, 팀 대시보드 |
| **15명+** | 부서 분화 | 부서별 뷰, 크로스팀 협업, 거버넌스 |

### ARR과 팀 규모의 상관관계 (B2B SaaS 기준) [^1]

| ARR | 마케팅 팀 규모 | 주요 역할 |
|-----|----------------|-----------|
| ~$5M | 1~3명 | 제너럴리스트 |
| $5~15M | 3~7명 | 콘텐츠, 디맨드젠, 오퍼레이션 분화 |
| $15~50M | 8~20명 | 채널별 전문가, 애널리스트 추가 |
| $50M+ | 20명+ | 브랜드, 프로덕트 마케팅, ABM 등 부서화 |

## 2.4 기술 역량과 데이터 인프라

### 기술 역량 수준

| 수준 | 특징 | 툴 니즈 |
|------|------|---------|
| **비개발자 팀** | 코드 작성 불가, GUI 의존 | 노코드, 드래그앤드롭, 템플릿 |
| **테크니컬 마케터 보유** | 간단한 스크립트, SQL 가능 | API 접근, 커스텀 이벤트, 쿼리 |
| **개발자 지원 가능** | 엔지니어 협업 | SDK, Webhook, 커스텀 연동 |
| **데이터팀 보유** | 데이터 웨어하우스, ETL | Reverse ETL, 데이터 싱크, 커스텀 모델 |

### 데이터 인프라 수준

| 수준 | 인프라 | 연동 방식 |
|------|--------|-----------|
| **초기** | 데이터 분산, 사일로 | 각 툴 자체 데이터 |
| **중기** | 기본 연동, 일부 통합 | 네이티브 연동, Zapier |
| **후기** | 데이터 웨어하우스 중심 | CDP, Reverse ETL, 실시간 파이프라인 |

# Part 3. 속성 조합과 니즈 분화

실제로는 영속적 속성과 변화하는 속성이 조합되어 고객사의 니즈를 형성한다.

## 예시: 동일 산업, 다른 성숙도

**이커머스 A사 (초기 스타트업)**
- 마케팅 담당 1명
- 기본 이메일 발송만 사용
- 니즈: 쉬운 장바구니 리마인드 설정, 템플릿

**이커머스 B사 (성장 단계)**
- 마케팅 팀 5명, 개발자 지원 가능
- 멀티채널 자동화 운영 중
- 니즈: 고급 세그먼트, A/B 테스트, 상품 추천 연동

**이커머스 C사 (대기업)**
- 마케팅 팀 30명, 데이터팀 별도
- AI 기반 개인화, 옴니채널
- 니즈: 엔터프라이즈 보안, 커스텀 ML 모델 연동

→ **같은 "이커머스"지만 완전히 다른 툴이 필요**

 모델   │          │                    │          │
   B2C  │ Mailchimp│    Klaviyo         │  Braze
        │ 기본     │    ActiveCampaign  │  + CDP
        │          │                    │          │
        └──────────┴────────────────────┴──────────┘

# 핵심 요약

## 왜 툴이 분화되는가?

1. **영속적 속성이 다르면** → 해결해야 할 문제 자체가 다름
   - B2B는 리드 전환, B2C는 리텐션
   - 이커머스는 구매, SaaS는 활성화, 앱은 습관화

2. **변화하는 속성이 다르면** → 같은 문제도 해결 방식이 다름
   - 초기는 "일단 되게", 후기는 "최적화"
   - 작은 팀은 "쉽게", 큰 팀은 "통제 가능하게"

3. **조합이 다르면** → 무수히 많은 세그먼트 발생
   - 같은 이커머스도 스타트업 vs 대기업은 다른 툴 필요
   - 같은 규모여도 B2B vs B2C는 다른 기능 필요
