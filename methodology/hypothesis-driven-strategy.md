# 전략 수립 방법론

> 리서치 완료 후, 전략/가설 도출 단계에서 실력있는 전략가(PO, PM, GTM)가 어떻게 사고하고 행동하는가

---

## 1. 핵심 원칙: "가설 먼저" (Hypothesis-Driven)

| 아마추어 | 프로 |
|----------|------|
| 데이터 다 모으고 → 분석하고 → 결론 도출 | **가설 먼저 세우고** → 검증에 필요한 데이터만 → 빠르게 판단 |

```
[열린 질문]
"CRM 마케팅 어떻게 할까?"

        ↓ 전환

[가설 형태]
"우리는 A 방향이 맞을 것이다. 왜냐하면 X, Y, Z 때문에."
```

---

## 2. 가설 도출 방법

### Step 1: 제약 조건 명확히 하기

| 질문 | 우리의 답 |
|------|-----------|
| 우리가 **잘하는 것**은? | (작성) |
| 우리가 **못하는 것**은? | (작성) |
| **시간/리소스** 제약은? | (작성) |
| **하면 안 되는 것**은? | (작성) |

### Step 2: "만약 ~라면" 시나리오 만들기

```
"만약 우리가 [Option A]에 올인한다면..."
→ 뭐가 좋아지고, 뭐가 나빠지는가?
→ 가장 큰 리스크는?
→ 성공하면 어떤 모습인가?

"만약 우리가 [Option B]로 간다면..."
→ ...
```

### Step 3: 가설 + 검증 기준을 세트로

```
가설: "[핵심 주장]"

검증 기준:
- 데이터: ...
- 고객 증거: ...
- 실험 결과: ...
```

---

## 3. 실력있는 전략가의 행동 패턴

| 단계 | 행동 | 산출물 |
|------|------|--------|
| **1. 가설 수립** | 3~5개 전략 옵션 + 각각의 가설 | 1-pager 가설 문서 |
| **2. 빠른 검증 설계** | 각 가설을 깨뜨릴 수 있는 질문 정의 | 검증 체크리스트 |
| **3. 최소 데이터 수집** | 필요한 것만, 빠르게 | 데이터 요청 (팀 협업) |
| **4. 고객 대화** | 5~10명과 직접 대화 | 인터뷰 노트 |
| **5. 스파링** | 동료/리더와 가설 부수기 | 정제된 가설 |
| **6. 의사결정 프레임** | "A vs B, 우리는 A를 선택한다. 왜냐하면..." | 의사결정 문서 |
| **7. 작게 실험** | 전체 베팅 전에 작은 실험 | MVP/파일럿 |

---

## 4. 가설 문서 템플릿

```markdown
## 가설: [제목]

### 핵심 주장
우리는 [A 방향]으로 가야 한다.

### 근거 (왜 이게 맞다고 생각하는가)
1. ...
2. ...
3. ...

### 리스크 (왜 이게 틀릴 수 있는가)
1. ...
2. ...

### 검증 방법 (어떻게 확인할 것인가)
- 데이터: ...
- 고객 인터뷰: ...
- 실험: ...

### 성공 시 모습 (맞다면 6개월 후 어떤 상태?)
- ...
```

## 6. 안티패턴 (피해야 할 것)

| 안티패턴 | 왜 문제인가 | 대신 |
|----------|-------------|------|
| "데이터 더 모으자" 무한 루프 | 결정 회피, 시간 낭비 | 가설 먼저, 필요한 데이터만 |
| 모든 옵션 열어두기 | 집중 부족, 리소스 분산 | 명확한 트레이드오프 인지 |
| 합의 기반 의사결정 | 타협안은 보통 최악 | 명확한 의사결정권자 + 근거 |
| 완벽한 계획 세우기 | 계획은 틀림, 실행이 중요 | 작은 실험으로 빠르게 검증 |
| 직관만으로 결정 | 편향, 사후 합리화 | 직관 + 데이터 균형 |
