## Executive Summary

채널톡 마케팅 기능에 대한 고객 인식을 4개 트랙(이탈, 사용현황, 온보딩, 신규기대)으로 분석한 결과, **마케팅 기능의 낮은 채택률은 기능 자체의 문제보다 구조적 장벽에 기인**함을 확인했습니다.

| 비용 민감도 | 25% | 이탈 논의의 1/4이 비용 관련 |

## CRM/마케팅 기대가 꺾이는 패턴

신규 고객의 39%가 CRM/마케팅에 관심을 갖고 도입하지만, 실제 사용 단계에서 기대가 꺾이는 구체적인 패턴:

### 신규 고객의 초기 기대

> "CRM·CS·마케팅을 동시에 개선할 수 있는 솔루션 검토"

> "리드/고객 DB 축적 → CRM 마케팅 가능 (유입을 상담으로만 소모하지 않고)"

> "CRM 기반 세그먼트 마케팅 필요성 인지"

### 기대가 꺾이는 5가지 패턴

| 패턴 | 고객 원문 |
|------|----------|
| **CS가 급해서 밀림** | "ai로 cs 효율화하는거 우선과제. 2차로 crm마케팅도 고도화 해보고자함. 근데 **우선순위는 alf임**" |
| **리소스 없음** | "1인 실무로 자사몰 운영/cs/채널톡 다 하고있어서 **cs리소스 빼는게 중요함**" |
| **성과가 안 보임** | "메시지를 발송해도 **매출 연계 효과가 직접적으로 보이지 않아** 우선순위에서 밀려 있음" |
| **세팅이 어려움** | "기본 세팅은 가능하나, **워크플로우 연결 및 고객 여정 설계 등 고도화 작업에 어려움**" |
| **상위 압박** | "대표님으로부터 **성과를 증명하지 못할 경우, 일반 상담 툴로 축소**하겠다는 통보" |

### 기대 → 꺾임의 타임라인

```
도입 시: "CRM 마케팅도 해보고 싶다" (39%)
    ↓
온보딩: "일단 CS/ALF 먼저" (58%)
    ↓
사용 중: "CS도 바쁜데 마케팅까지 손 못 댐" (62%)
    ↓
갱신 시: "마케팅 안 써서 빼겠다" (7%)
```

## CS-마케팅 연결 고리

"CS 세팅할 때 마케팅도 같이 하자"는 접근은 고객 입장에서 연관성이 낮게 느껴질 수 있음.
실제 미팅 로그에서 발견한 **자연스러운 연결 고리**:

### CS 데이터 → 마케팅 활용

| CS 관점 | 연결점 | 마케팅 활용 |
|---------|--------|------------|
| 상담 데이터 축적 | → | "유입을 상담으로만 소모하지 않고 DB확보 → 전환/재방문 유도" |
| 단순 문의 자동화 | → | "인력을 줄이기보다, 더 중요한 업무(마케팅)에 집중" |
| 고객 정보 수집 | → | "고객정보 연동 및 세그먼트 분류를 통한 마케팅 자동화" |
| 장바구니 이탈 문의 | → | "장바구니 이탈 고객 리마인드 캠페인" |

### 현실적인 접근법

**"새로운 마케팅"이 아닌 "CS의 연장선"으로 포지셔닝**

| 기존 접근 | 개선 접근 |
|----------|----------|
| "마케팅 기능도 써보세요" | "CS에서 수집된 데이터로 자동 리마인드" |
| "캠페인 세팅하세요" | "교환/반품 완료 고객 → 2주 후 리뷰 요청" |
| "세그먼트 만드세요" | "VIP 고객 상담 시 자동 태그 → 전용 혜택 안내" |

**핵심**: CS 워크플로우 세팅 시 자연스럽게 수집되는 데이터를 마케팅에 연결하는 **CS 후속 캠페인**부터 시작

## 개선 방향 제안

### 1. 온보딩 단계에서 마케팅 함께 세팅

| 현재 | 개선 |
|------|------|
| CS 세팅 → (나중에) 마케팅 | CS + 마케팅 동시 세팅 |
| 고객이 직접 세팅 | 온보딩 전담팀이 기본 캠페인 세팅 |
| 마케팅은 옵션 | 마케팅 기본 템플릿 자동 활성화 |

### 2. 리소스 장벽 해소

| 현재 | 개선 |
|------|------|
| 세팅 복잡 | 원클릭 캠페인 템플릿 |
| 방법 모름 | 인앱 가이드/튜토리얼 |
| 1인 실무자 한계 | AI 기반 캠페인 자동 추천 |

### 3. 가치 증명 자동화

| 현재 | 개선 |
|------|------|
| ROI 체감 어려움 | 자동 성과 리포트 |
| 전환율 불확실 | 매출 연동 대시보드 |
| "마케팅 효과 없다" | 캠페인별 기여도 시각화 |

### 4. 비용 인식 개선

| 현재 | 개선 |
|------|------|
| "가장 비싼 SaaS" | 절감 효과 대비 비용 시각화 |
| 변동 비용 예측 불가 | 월별 예상 비용 대시보드 |
| 마케팅만 해지 시도 | 마케팅 ROI 증명으로 방어 |

## 결론

채널톡 마케팅 기능의 낮은 채택률은 **"기능의 문제"가 아닌 "구조의 문제"**입니다.

신규 고객의 39%가 CRM/마케팅에 관심을 갖고 도입하지만, 온보딩 단계에서 CS/ALF 중심으로 진행되고(58%), 사용 단계에서는 CS 우선으로 밀리며(62%), 갱신 시점에는 가치 미입증으로 해지/축소 대상이 됩니다.

이 구조를 깨기 위해서는 **온보딩 단계에서 마케팅을 함께 세팅**하고, **리소스 장벽을 낮추며**, **가치를 자동으로 증명**하는 시스템이 필요합니다.
